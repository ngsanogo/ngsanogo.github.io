Je m'appelle Issa Sanogo. Je travaille dans la data depuis neuf ans — la donnée clinique à l'AP-HP, la recherche médicale à l'Institut Jérôme Lejeune, le SaaS aujourd'hui. Ce blog est mon carnet d'ingénierie : j'écris quand quelque chose me marque dans mon travail, un problème que j'aurais voulu trouver documenté, une conviction que j'ai envie de défendre.

Ma thèse : **les plateformes data fiables se construisent par l'explicité, pas par la sophistication.** Un pipeline dont on comprend le comportement en lisant le code est un pipeline bien écrit. La complexité technique est souvent un symptôme, rarement une solution.

## Les articles pour commencer

Ces quatre articles résument le mieux comment je pense les plateformes data :

- [Le Zen du data engineering](/posts/zen-of-data-engineering/) — les principes qui séparent un pipeline fragile d'un pipeline fiable. Mon manifeste.
- [Construire un pipeline production-grade](/posts/pipeline-production-grade/) — idempotence, gestion des erreurs, traitement incrémental. Les trois propriétés non négociables.
- [Qualité des données en santé : pourquoi c'est plus difficile qu'ailleurs](/posts/data-quality-healthcare-why-harder/) — neuf ans de terrain condensés. L'article que personne d'autre ne peut écrire à ma place.
- [Les 6 dimensions de la qualité des données](/posts/data-quality-fundamentals/) — le cadre conceptuel avant la mise en pratique.
